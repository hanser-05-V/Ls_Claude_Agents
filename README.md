## 协作协议 v2.0 — 核心

**思考在主窗口，执行丢给子Agent。**

| 模式 | 场景 | 上下文丢失 | 方式 |
|------|------|:--:|------|
| 主窗口扮演 | 讨论/分析/设计/审查/教学 | **0%** | 主脑读取Agent配置，切换角色口吻 |
| 子Agent自包含 | 写代码/查Bug/批量操作 | **≤15%** | 精读源码→构造完整自包含prompt→注入 |
| 两阶段审查 | Spec合规 + 代码质量 | 独立审 | ①Spec审查 → ②代码质量审查，不信任实施者报告 |
| 链式交接 | 设计→编码→审查多部门 | 10-15% | 每环节留交接文件，下环节先读再开工 |

**子Agent铁律：**
- 自包含Prompt（任务全文+代码片段+设计约束+验收标准）
- Fresh Agent Per Task（一个Agent只做一件事）
- 审查不信任原则（审查者独立读代码验证）
- 升级不忽略（BLOCKED → 分析原因 → 升级模型/拆分/报告帅老大）

详见：`docs/协作协议_v2.0.md`

## 使用

### 启动
```
Ls
```

### @ 直接指派（主窗口扮演模式，零上下文丢失）
```
@架构导师 解释这个模块的设计
@小爱 各部门进度汇报
@文档书记 记笔记
```

### 派子Agent（纯执行任务，≤15%丢失）
```
派子Agent排查 login.lua 闪退
```

### 查看
```
任务墙
文档书记，记忆库有哪些规则？
```

### 关闭
```
关闭部门
```
