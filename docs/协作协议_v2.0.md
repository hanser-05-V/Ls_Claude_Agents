## 四种协作模式

| 模式 | 场景 | 上下文丢失 | 方式 |
|------|------|:--:|------|
| **主窗口扮演** | 讨论/分析/设计/审查/教学 | **0%** | 主脑读取对应 agent 配置，切换角色口吻 |
| **子 Agent 自包含** | 写代码/查Bug/批量操作 | **≤15%** | 控制器精读源码→构造完整自包含 prompt→注入 |
| **两阶段审查** | Spec合规 + 代码质量验证 | 独立子agent各审各的 | ①Spec审查子agent → ②代码质量审查子agent |
| **链式交接** | 设计→编码→审查多部门 | 10-15% | 每环节留交接文件，下环节先读再开工 |

## 核心原则

**思考在主窗口，执行丢给子 agent。**

## 子 Agent 铁律

1. **自包含 Prompt**：控制器先精读源码，再构造完整 prompt（任务全文+代码片段+设计约束+验收标准）
2. **Fresh Agent Per Task**：一个子 agent 只做一件事，做完即关
3. **审查不信任原则**：审查者独立读代码验证，不信任实施者自述
4. **两阶段审查**：Spec合规审查先于代码质量审查，顺序不可颠倒
5. **升级不忽略**：子 agent 报告 BLOCKED/NEEDS_CONTEXT 时必须响应，不原样重试

## 审查循环

```
实施者完成 → ①Spec审查(必须独立验证) → ②代码质量审查 → 任务完成
              ❌不合格→修复→re-review    ❌不合格→修复→re-review
```

## 模型策略

|---------|------|
| 机械执行 | 便宜快速 |
| 集成/调试 | 标准 |
| 架构/设计/审查 | 最强 |
